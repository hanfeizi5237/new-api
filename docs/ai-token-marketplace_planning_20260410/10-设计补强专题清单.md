# 10-设计补强专题清单

更新时间：2026-04-12

## 1. 目的

当前 `01-09` 文档已经足够支撑项目进入 `M1`，但为了降低后续返工风险，建议继续补齐若干“专题级设计文档”。

这些专题不再讨论“是否要做”，而是专门回答“具体怎么做、做到什么粒度、冻结哪些实现口径”。

## 3. 各专题建议补强内容

### 3.1 `SellerKeyVault 与密钥生命周期设计`

当前缺口：

- 只定义了 `supply_account` 和绑定关系
- 未冻结密钥密文存储、解密边界、访问审计、轮换和吊销逻辑

需要回答：

- 卖家原始 Key 是否继续存入 `channels.key`
- 是否引入独立密钥表
- 应用侧加密的主密钥从哪里来
- 哪些角色可以看到明文，哪些只能看到脱敏值
- 验活、轮换、失效、封禁、恢复如何流转

### 3.2 `支付订单与权益发放时序设计`

当前缺口：

- 已有订单状态轴，但还没有“下单到支付回调到发权”的完整时序

需要回答：

- 创建订单后什么时候锁库存
- 同步返回与异步回调如何协同
- 重复回调、乱序回调、失败回调如何处理
- entitlement 什么时候发放，失败后如何补偿

### 3.3 `entitlement 路由分配与扣减算法设计`

当前缺口：

- 已说明按 lot 扣减，但没冻结分配算法与责任归属

需要回答：

- 一个请求是否允许跨 lot、跨 seller、跨 supply
- retry 后是否允许更换 seller
- 流式中途中断如何计算实际扣减
- sticky seller / sticky supply 是否需要
- usage 与 settlement 如何对应到唯一卖家责任主体

### 3.4 `库存口径、超卖控制与自动停售规则设计`

当前缺口：

- `supply_account` 与 `inventory_snapshot` 的字段已有，但口径公式未冻结

需要回答：

- 可售库存的计算公式
- 风险折扣如何影响可售值
- 下单时锁多少、什么时候释放
- 超卖时谁优先承担损失
- 自动停售、人工恢复和库存回补条件

### 3.5 `Token 与 Entitlement 绑定及资金源优先级设计`

当前缺口：

- 还没有冻结 `Token` 在 entitlement 场景里到底是认证载体还是计费载体
- 也没有冻结 entitlement 与现有 `wallet/subscription` 的优先级

需要回答：

- entitlement 是绑定 `user` 还是 `token`
- 同一用户多个 token 是否共享 entitlement
- 命中 entitlement 时是否还扣 token quota
- entitlement 不足时是否允许回落到 wallet/subscription
- relay 里是否要增加新的 billing source

### 3.6 `结算、佣金、退款与冲销公式设计`

当前缺口：

- `settlement_entry` 已存在，但结算确认点未冻结

需要回答：

- 毛收入、平台抽佣、卖家净收入如何计算
- 是按支付确认还是按 usage 消耗确认收入
- 部分退款、争议退款、卖家赔付如何冲销
- 保证金与待结算余额如何联动

### 3.7 `异步事件、任务重试与补偿机制设计`

当前缺口：

- 已经知道很多动作需要异步，但还没冻结事件模型

需要回答：

- 首期到底用定时任务、Redis Streams 还是其他方案
- 事件清单、重试次数、死信处理怎么做
- 哪些动作必须同步，哪些动作允许最终一致性

### 3.8 `API 契约与错误码规范`

当前缺口：

- 已列出接口族，但没有冻结 DTO、错误码、权限与幂等约定

需要回答：

- DTO 字段命名
- 分页格式
- 业务错误码
- 幂等头/参数约定
- 管理端、卖家端、买家端接口边界

### 3.9 `RBAC、敏感操作与审计权限矩阵设计`

当前缺口：

- 现有项目有 `UserAuth/AdminAuth/RootAuth`，但集市域权限没展开

需要回答：

- 谁能创建 seller
- 谁能审核 listing
- 谁能查看密钥明文
- 哪些操作必须二次验证
- 哪些行为必须留审计日志

### 3.10 `信息架构与关键页面流转设计`

当前缺口：

- 已有后台页面任务书，但没有页面流、空状态、异常状态设计

需要回答：

- 买方集市首页、商品详情、下单、支付结果、我的订单怎么串起来
- 卖家与平台后台审核流程如何展示
- 风险提示和异常反馈放在哪里

### 3.11 `Pilot 灰度、对账、止损与回滚 Runbook`

当前缺口：

- 有风险清单，没有独立上线操作手册

需要回答：

- 白名单卖家如何开通
- 上线检查项有哪些
- 对账不平时如何止损
- Key 集体失效时谁来停卖
- 如何回滚到“仅平台自营供给”模式
