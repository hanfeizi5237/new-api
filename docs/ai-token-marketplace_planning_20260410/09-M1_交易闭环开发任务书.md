# M1 交易闭环 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `new-api-main` 中落地卖家供给、商品上架、买方下单支付、权益发放、平台代理调用的 M1 最小闭环。

**Architecture:** 保持现有 `router -> controller -> service -> model` 单体分层，新增 marketplace 交易域，并以 `Channel/Ability/Relay` 作为运行时内核。M1 不拆微服务，不引入 MQ 硬依赖，不重写现有 relay，只在关键节点做插点式接入。

**Tech Stack:** Go 1.22+, Gin, GORM v2, React 18, Semi UI, Bun, SQLite/MySQL/PostgreSQL, Redis（可选增强）。

**Frozen M1 Boundaries:** 仅支持 `text/chat` 模型、按 `token` 计量的 entitlement；卖家原始凭证主事实源为 `seller_secret`；单次请求只允许命中一种最终资金来源，不允许 entitlement 与 wallet/subscription 混合扣费。

## 2. 交付范围

M1 只交付以下闭环：

1. 管理员可创建/审核卖家与商品
2. 管理员可导入、验证、禁用卖家凭证并同步到运行时 channel
3. 买方可浏览商品、下单、支付、查看订单
4. 支付成功后生成 entitlement
5. 买方继续使用现有平台 `Token` 调用模型
6. relay 能按 entitlement 约束选路并扣减权益
7. 管理端可查看 sellers/listings/orders/entitlements

M1 不包含：

- 争议工单闭环
- 结算释放
- 自助提现
- 信用分
- 复杂风控
- 图像/音频/task 类型 entitlement
- 单请求内 entitlement 与 wallet/subscription 混合计费
